# Доступ к BASE из России

## Проблема

Если сайт BASE не открывается на Mac (или других устройствах) в России, это связано с блокировками интернет-трафика на стороне российских провайдеров, а не с ограничениями Vercel.

## Причины блокировки

1. **DPI (Deep Packet Inspection)** - провайдеры анализируют трафик и могут блокировать запросы к определенным сервисам
2. **Блокировка IP-адресов** - IP-адреса Vercel могут быть заблокированы
3. **Блокировка доменов** - домены `*.vercel.app` могут быть в списке блокировок

## Решения

### 1. Использование VPN (рекомендуется)

Если VPN не работает, попробуйте VPN-сервисы с продвинутыми протоколами:
- **VLESS с TLS/XTLS**
- **Shadowsocks с обфускацией**
- **WireGuard с Cloak/Obfs/Masking**
- **TLS over WebSocket через порт 443**

### 2. Изменение DNS-серверов

Попробуйте использовать публичные DNS-серверы:
- **Cloudflare**: `1.1.1.1` и `1.0.0.1`
- **Google**: `8.8.8.8` и `8.8.4.4`
- **Quad9**: `9.9.9.9` и `149.112.112.112`

**На Mac:**
1. Системные настройки → Сеть
2. Выберите ваше подключение → Дополнительно
3. Вкладка DNS → Добавьте DNS-серверы выше

### 3. Очистка кэша браузера

1. **Safari**: Safari → Настройки → Конфиденциальность → Управление данными веб-сайтов → Удалить все
2. **Chrome**: Chrome → Настройки → Конфиденциальность → Очистить данные просмотра

### 4. Использование кастомного домена

Если у вас есть кастомный домен, настройте его в Vercel:
1. Перейдите в настройки проекта на Vercel
2. Добавьте кастомный домен
3. Настройте DNS-записи

Кастомный домен может помочь обойти блокировки, так как он не будет содержать `vercel.app` в названии.

### 5. Проверка через мобильный интернет

Попробуйте подключиться через мобильный интернет (4G/5G) вместо Wi-Fi - иногда мобильные операторы имеют другие настройки блокировок.

## Диагностика проблемы

### Проверка доступности сайта

```bash
# Проверка DNS
nslookup your-app.vercel.app

# Проверка доступности
curl -I https://your-app.vercel.app

# Проверка через другой DNS
dig @1.1.1.1 your-app.vercel.app
```

### Проверка в браузере

1. Откройте **Developer Tools** (F12)
2. Перейдите на вкладку **Network**
3. Попробуйте загрузить сайт
4. Проверьте, какие запросы блокируются или зависают

### Типичные ошибки

- **ERR_CONNECTION_TIMED_OUT** - таймаут соединения (возможна блокировка)
- **ERR_NAME_NOT_RESOLVED** - проблема с DNS
- **ERR_CONNECTION_REFUSED** - соединение отклонено
- **Бесконечная загрузка** - возможна блокировка на уровне DPI

## Альтернативные решения

Если ничего не помогает, можно:
1. Использовать прокси-сервер
2. Настроить SSH-туннель
3. Использовать мобильный интернет с роумингом
4. Обратиться к провайдеру с запросом о разблокировке

## Техническая информация

- **Регион Vercel**: `fra1` (Франкфурт, Германия)
- **Edge Functions**: Работают в указанном регионе
- **CDN**: Vercel использует глобальную CDN сеть

## Поддержка

Если проблема сохраняется, пожалуйста:
1. Соберите информацию о вашем провайдере
2. Проверьте логи браузера (Console и Network)
3. Попробуйте разные браузеры
4. Сообщите о проблеме с деталями
